Pendahuluan

Tujuan

Dokumen ini dibuat untuk memberikan gambaran secara menyeluruh mengenai perancangan dan pengembangan sistem aplikasi Pabrik Tempe berbasis web. Tujuan utama dari pembuatan sistem ini adalah untuk membantu proses pengelolaan usaha pabrik tempe agar menjadi lebih efektif, efisien, dan terorganisir.

Selain itu, sistem ini juga bertujuan untuk:

Mempermudah proses pencatatan produksi tempe
Mengelola data stok bahan baku dan produk jadi secara terstruktur
Membantu pencatatan transaksi penjualan secara otomatis
Menyediakan laporan yang akurat dan mudah dipahami
Mengurangi kesalahan dalam pencatatan manual

Dengan adanya sistem ini, diharapkan proses bisnis pada pabrik tempe dapat berjalan lebih baik dan mendukung pengambilan keputusan yang lebih tepat.


Ruang Lingkup

Ruang lingkup dari sistem aplikasi Pabrik Tempe ini mencakup beberapa proses utama dalam kegiatan usaha, yaitu:

Pengelolaan Produksi
Sistem digunakan untuk mencatat jumlah produksi tempe setiap hari, termasuk data bahan baku yang digunakan.
Pengelolaan Stok
Sistem mencatat ketersediaan bahan baku dan produk jadi, sehingga memudahkan dalam monitoring stok.
Sistem Penjualan
Digunakan untuk mencatat transaksi penjualan produk kepada pelanggan.
Laporan
Menyediakan laporan produksi dan penjualan dalam periode tertentu.
Pemesanan Produk
Pelanggan dapat melakukan pemesanan produk melalui sistem.

Sistem ini dibatasi hanya untuk kebutuhan pengelolaan internal dan belum mencakup integrasi dengan sistem eksternal seperti pembayaran online atau logistik secara otomatis.

Aplikasi Pabrik Tempe merupakan sistem informasi berbasis web yang dirancang untuk membantu proses bisnis pada usaha produksi tempe. Sistem ini berfungsi sebagai alat bantu digital yang menggantikan proses manual dalam pencatatan produksi, pengelolaan stok, hingga transaksi penjualan.

Sebelumnya, proses bisnis pabrik tempe masih dilakukan secara manual seperti pencatatan di buku, yang berpotensi menimbulkan kesalahan, kehilangan data, serta kesulitan dalam pencarian informasi. Oleh karena itu, aplikasi ini dikembangkan untuk meningkatkan efisiensi, akurasi, dan kemudahan dalam pengelolaan data.

Sistem ini memiliki dua jenis pengguna utama, yaitu:

Admin, yang bertugas mengelola seluruh data dalam sistem seperti produksi, stok, dan laporan.
Pelanggan, yang dapat melakukan pemesanan produk secara langsung melalui sistem.

Aplikasi ini juga dirancang agar dapat dikembangkan lebih lanjut, seperti integrasi dengan sistem pembayaran online, pengiriman barang, serta pengembangan ke platform mobile (Android).

Software Interface

Aplikasi ini menggunakan antarmuka berbasis web (web interface) yang dapat diakses melalui browser. Interface dirancang agar sederhana, mudah digunakan (user-friendly), dan responsif.

Beberapa bagian utama tampilan sistem meliputi:

Halaman Login & Register
Digunakan untuk autentikasi pengguna sebelum masuk ke sistem.
Dashboard
Menampilkan ringkasan data seperti jumlah produksi, stok, dan penjualan.
Halaman Produksi
Digunakan untuk mencatat data produksi tempe setiap hari.
Halaman Stok
Menampilkan data bahan baku dan produk jadi yang tersedia.
Halaman Penjualan
Digunakan untuk mencatat transaksi penjualan.
Halaman Laporan
Menampilkan laporan data penjualan dan produksi dalam periode tertentu.

Aplikasi dapat diakses menggunakan berbagai browser seperti:

Google Chrome
Microsoft Edge
Mozilla Firefox
2.3 Hardware Interface

Sistem ini tidak memerlukan perangkat keras khusus, namun dapat berjalan dengan spesifikasi minimum sebagai berikut:

Perangkat Pengguna (Client):
Laptop atau komputer dengan RAM minimal 4GB
Smartphone (opsional untuk akses mobile)
Perangkat Server:
Server lokal menggunakan XAMPP
Database MySQL

Sistem ini berjalan pada arsitektur client-server, di mana pengguna mengakses aplikasi melalui browser dan data diproses oleh server.

2.4 Manfaat Produk

Aplikasi Pabrik Tempe memberikan berbagai manfaat, antara lain:

Efisiensi Waktu
Proses pencatatan produksi dan penjualan menjadi lebih cepat dibandingkan cara manual.
Mengurangi Kesalahan (Human Error)
Data tersimpan secara otomatis sehingga mengurangi kesalahan pencatatan.
Pengelolaan Data Lebih Terstruktur
Semua data tersimpan dalam database sehingga mudah dicari dan diolah.
Monitoring Real-Time
Pemilik usaha dapat memantau kondisi produksi dan penjualan kapan saja.
Meningkatkan Produktivitas Usaha
Dengan sistem yang terorganisir, proses bisnis menjadi lebih optimal.
2.5 Karakteristik User

Pengguna sistem dibagi menjadi beberapa kategori:

1. Admin
Memiliki akses penuh ke sistem
Mengelola data produksi, stok, dan penjualan
Mengakses laporan
2. Pemilik Usaha
Melihat laporan produksi dan penjualan
Memantau perkembangan usaha
3. Pelanggan
Melakukan pemesanan produk
Melihat informasi produk

Karakteristik umum pengguna:

Memiliki pengetahuan dasar penggunaan komputer
Dapat mengoperasikan browser
